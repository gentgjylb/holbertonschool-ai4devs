# AI Fixes Applied

This document logs all decisions made concerning the AI-provided recommendations. Included are heavily detailed justifications to establish extreme clarity on why each framework rule was either rejected or accepted. Additionally, the unit tests inside `test_tasks.py` have been explicitly expanded to validate that the accepted logical fixes perform flawlessly.

| AI Suggestion                                | Action Taken     | Notes                    |
|----------------------------------------------|------------------|--------------------------|
| Add input sanitization to escape HTML for `title` and `description` | ❌ Rejected      | While injection vulnerabilities are a serious threat, input escaping code belongs exclusively at the frontend presentation layer. Modifying raw data on the backend model breaks clean architecture boundaries (separation of concerns) and corrupts raw data state. |
| Avoid unconditionally trusting `uuid.uuid4()` for cryptographical bounds | ❌ Rejected      | This project uses `uuid.uuid4()` strictly as an internal memory identification mechanism to lookup array objects locally. The system has zero authorization elements and never issues secure-access tokens, rendering this specific cryptographic replacement suggestion unneeded. |
| Decouple heavy filtering bounds into a Fluent Query Builder architecture | ❌ Rejected      | Building a completely decoupled query builder framework represents major structural over-engineering for a static, memory-bound local list manager. We abide by the YAGNI principle here to prioritize maintainability over abstract theoretical patterns. |
| Explicitly document lowercase formatting behavior inside `add_tag(tag)` | ✅ Applied       | The docstring was natively updated. Expanding the documentation eliminates API ambiguity and allows any client to know exactly why their `"URGENT"` tag resolves to `"urgent"`. Validated natively in `test_task_tags_formatted_lowercase`. |
| Employ timezone-aware markers prior to `isoformat()` instantiation | ✅ Applied       | Using native `datetime.now(timezone.utc)` eliminates severe timestamp errors occurring when accessing dates across multiple international server locations. Tested explicitly inside `test_task_creation`. |
| Flatten `filter_tasks()` consecutive iterations resolving O(N * K) metrics | ✅ Applied       | The nested series of 5 generator arrays was entirely consolidated into a master loop structure. This boosts querying speeds natively and guarantees absolute `O(N)` bounds. Covered efficiently through existing `filter_tasks` unittests ensuring matching outputs. |
| Extract hardcode limits to leveraging a formal `Enum` class in `sort_tasks` | ✅ Applied       | Developed an isolated `SortAttribute` Enum class. By natively trying to parse string attributes into the Enum dictionary, we gracefully block unsupported arguments while retaining full backward compatibility for older strings. Validated through `test_sort_tasks_enum_logic`. |
| Instantiate discrete scalar values inside a single loop sweep for `summary()` | ✅ Applied       | Reduced a triple aggregation sequence into exclusively traversing `self.tasks` exactly one time. Iteratively accumulating values on the fly massively eliminates processing pipeline memory drain. Validated by `test_count_overdue_and_summary`. |
| Introduce a lightweight `count_overdue()` isolated date-resolving iterator | ✅ Applied       | Formally deployed a single localized iterator summing exclusively pending assignments expiring prior to today. Prevents dynamic array memory expansion overhead completely. Validated via `test_count_overdue_and_summary`. |
