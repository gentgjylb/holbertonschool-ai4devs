# Data Model - AI-Powered Startup MVP Builder

## Overview

The data model is designed around a **session-scoped project** where no user authentication
is required. Each browser session owns one active project. A project contains features and
user stories generated by the AI, along with a record of any exports produced.

---

## Entities

### 1. Session

Represents an anonymous browser session. Acts as the top-level owner of all data in the
MVP phase.

| Field | Type | Constraints | Description |
|---|---|---|---|
| `id` | UUID | PRIMARY KEY | Unique session identifier (set as a browser cookie). |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW | When the session was first created. |
| `expires_at` | TIMESTAMP | NOT NULL | Expiry time (e.g., 24 hours after creation). |
| `language` | VARCHAR(10) | NOT NULL, DEFAULT 'en' | Language code for the session (MVP: English only). |

---

### 2. Project

Represents a single MVP concept being built within a session. A session can have one
active project at a time (expanded to multiple in future versions).

| Field | Type | Constraints | Description |
|---|---|---|---|
| `id` | UUID | PRIMARY KEY | Unique project identifier. |
| `session_id` | UUID | FOREIGN KEY → Session.id | Owning session. |
| `title` | VARCHAR(255) | NOT NULL | Short name for the project/startup idea. |
| `raw_idea` | TEXT | NOT NULL | The original plain-language description entered by the user. |
| `problem_statement` | TEXT | NULLABLE | AI-extracted problem statement. |
| `target_users` | TEXT | NULLABLE | AI-extracted target user description. |
| `constraints` | TEXT | NULLABLE | User-defined constraints (time, budget, team size). |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW | When the project was created. |
| `updated_at` | TIMESTAMP | NOT NULL | Last modification timestamp. |

---

### 3. Feature

Represents a single product feature suggested or confirmed for an MVP project. Limited to
10 features per project.

| Field | Type | Constraints | Description |
|---|---|---|---|
| `id` | UUID | PRIMARY KEY | Unique feature identifier. |
| `project_id` | UUID | FOREIGN KEY → Project.id | Owning project. |
| `title` | VARCHAR(255) | NOT NULL | Short feature name (e.g., "AI-Guided Idea Intake"). |
| `description` | TEXT | NOT NULL | Detailed description of what the feature does. |
| `priority` | ENUM | NOT NULL | MoSCoW value: `must_have`, `should_have`, `could_have`, `wont_have`. |
| `in_scope` | BOOLEAN | NOT NULL, DEFAULT TRUE | Whether the feature is within the stated constraints. |
| `ai_suggested` | BOOLEAN | NOT NULL, DEFAULT TRUE | Whether the feature was AI-generated (`true`) or manually added (`false`). |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW | When the feature was added. |

---

### 4. UserStory

Represents a user story auto-generated from a scoped feature.

| Field | Type | Constraints | Description |
|---|---|---|---|
| `id` | UUID | PRIMARY KEY | Unique user story identifier. |
| `feature_id` | UUID | FOREIGN KEY → Feature.id | The feature this story was derived from. |
| `project_id` | UUID | FOREIGN KEY → Project.id | Owning project (denormalized for query convenience). |
| `role` | VARCHAR(100) | NOT NULL | The user role (e.g., "startup founder"). |
| `action` | TEXT | NOT NULL | What the user wants to do. |
| `benefit` | TEXT | NOT NULL | Why the user wants it. |
| `full_text` | TEXT | NOT NULL | Assembled story: "As a [role], I want to [action] so that [benefit]." |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW | When the story was generated. |

---

### 5. ExportRecord

Tracks each ZIP export generated for a project, enabling re-download within the session.

| Field | Type | Constraints | Description |
|---|---|---|---|
| `id` | UUID | PRIMARY KEY | Unique export identifier. |
| `project_id` | UUID | FOREIGN KEY → Project.id | Owning project. |
| `file_path` | VARCHAR(512) | NOT NULL | Path or key to the ZIP file in object storage. |
| `file_size_bytes` | INTEGER | NULLABLE | Size of the generated archive in bytes. |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT NOW | When the export was generated. |
| `downloaded_at` | TIMESTAMP | NULLABLE | When the user last downloaded the file. |

## Entity Relationship Diagram

```
Session
  │
  │ 1:1 (MVP phase)
  ▼
Project ──────────────────────────────┐
  │                                   │
  │ 1:N (max 10)                      │ 1:N
  ▼                                   ▼
Feature                         ExportRecord
  │
  │ 1:N
  ▼
UserStory
```

---

## Relationships Summary

| From | To | Cardinality | Description |
|---|---|---|---|
| Session | Project | 1 : 1 (MVP) | One session owns one active project. |
| Project | Feature | 1 : N (≤ 10) | A project has up to 10 features. |
| Project | ExportRecord | 1 : N | A project can have multiple export snapshots. |
| Feature | UserStory | 1 : N | Each in-scope feature generates one or more user stories. |
