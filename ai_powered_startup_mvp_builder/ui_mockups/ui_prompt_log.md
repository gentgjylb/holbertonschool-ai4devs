# UI Pattern Prompt Log

This document tracks the prompt iterations used to generate the UI mockups for the AI-Powered Startup MVP Builder.

## 1. Idea Intake Screen
- **Initial Prompt**: A sleek, modern web application UI design for an AI-powered startup builder. The screen shows an 'Idea Intake' form where a user can type their startup idea into a large text area. To the side, there is an AI chat panel analyzing the input. Dark mode, vibrant purple and blue accents, high quality, Dribbble style, UI/UX design, clean interface.
- **Screenshot**: `./idea_intake_screen_1776624446170.png`
- **Revision**: The initial result provided a clean and clear Dribbble-style UI with vibrant purple accents that perfectly match the tech startup theme. No further revisions were necessary as the interface met the MVP requirement for text-based analysis tools.

## 2. Feature Prioritization Screen
- **Initial Prompt**: A modern UI/UX design of a web application for startup planning. The screen shows a 'Feature Prioritization' dashboard using the MoSCoW method. There are four columns or sections labeled 'Must-have', 'Should-have', 'Could-have', and 'Won't-have', with colorful cards representing features. Modern, dark mode, tech startup aesthetic, high resolution, Dribbble style.
- **Screenshot**: `./feature_prioritization_screen_1776625090657.png`
- **Revision**: The generation process initially failed with a 503 error from the image generation server, but upon retry it succeeded, capturing the kanban-board MoSCoW format well. The column layout makes the features easily digestible for non-technical founders.

## 3. Document Generator Screen
- **Initial Prompt**: A clean, professional modern web UI design for a document generator. The screen displays a two-pane layout: on the left, a list of MVP features and User Stories, and on the right, a live preview of a generated markdown document titled 'MVP Concept'. Tech interface, dark mode, blue and green accents, UI mockup, high quality, web app.
- **Screenshot**: `./document_generator_screen_1776625060501.png`
- **Revision**: The resulting output correctly implemented the side-by-side pane layout allowing the user to view accepted user stories while reading the output Markdown document. It integrates well into the rest of the application flow.
